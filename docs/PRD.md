| 항목 | 내용 |

---

### 1.2 배경 및 문제 정의

기존 시중의 뉴스 감성 분석 도구들은 아래 두 가지 구조적 한계를 가진다.

1. **노이즈 취약성**: 단순 키워드 빈도 기반 분석은 시장의 일시적 과잉 반응을 걸러내지 못함.
2. **도메인 무지**: 반도체 업계 고유 용어(감산, HBM, 수율 등)를 범용 감성 사전으로 처리하여 신호의 신뢰도가 낮음.

`Semi Senti`는 이 두 한계를 동시에 해결한다. 반도체 도메인 특화 NLP 사전으로 노이즈를 필터링하고, 재무 펀더멘털 밴드를 1차 필터로 삼아 감성 신호의 허위 양성(False Positive)을 억제한다.

### 2.2 핵심 사용자 시나리오

#### 시나리오 1 — 종목 분석 조회 (핵심 플로우)

```
사용자 → 대시보드 접속
       → 반도체 종목 선택 (예: SK하이닉스)
       → 시스템: 캐시 확인 → (미스 시) API 실시간 수집 → NLP 분석 실행
       → 화면: 캔들차트 + 매매 시그널 마커 + 감성 게이지 + 재무 요약 렌더링
       → 사용자: 현재 구간이 '저평가+공포'임을 확인 → 매수 판단 보조
```

#### 시나리오 2 — 다이버전스 포착 (기회 발굴)

```
주가 하락 중 → DivergenceDetector 감지:
              뉴스 감성 지수는 반등 신호
       → 대시보드에 '괴리 주의' 마커 표시
       → 사용자: 추세 반전 가능성 인지 → 포지션 재검토
```

#### 시나리오 3 — 알림 수신 (비실시간 사용자)

```
시스템: 감성 점수 < -70 AND 현재가 < 밴드 하단 조건 충족
      → NotificationManager: 텔레그램 메시지 발송
      → 사용자: 앱 미실행 상태에서도 매수 타점 알림 수신
```

#### F-2.1 / F-2.2 반도체 특화 NLP 감성 스코어링

**처리 파이프라인:**

1. **형태소 분석 (Tokenize)**  
   KoNLPy 라이브러리로 뉴스 본문·헤드라인에서 명사·형용사 추출

2. **키워드 매칭 (Matching)**  
   추출된 단어를 자체 구축 '반도체 특화 사전'과 대조  
   도메인 고유 해석 예시: `감산` → 호재(양수 가중치), `재고` 문맥 → 악재(음수 가중치)

3. **점수 산출 (Calculation)**

$$\text{Raw Score} = \sum_{i=1}^{n} w_i \quad (w_i: \text{각 매칭 키워드의 가중치})$$

4. **정규화 (Normalization)**  
   원시 점수를 `-100 ~ +100` 범위로 정규화  
   `-100`: 극단적 부정(패닉), `0`: 중립, `+100`: 극단적 긍정(환희)

5. **저장**  
   일자별·종목별 감성 지수를 SQLite에 기록

#### F-3.1 / F-3.2 다중 요인 기반 매매 시그널 도출

**매매 시그널 산출 로직:**

```
입력:
  - 현재가 (Current Price)
  - 펀더멘털 적정 가치 밴드 하단 (Band_Low)
  - 펀더멘털 적정 가치 밴드 상단 (Band_High)
  - 감성 점수 (Sentiment Score: -100 ~ +100)

로직:
  IF   현재가 < Band_Low  AND  Sentiment Score < -70
  THEN → 매수(BUY)   ← 저평가 구간 + 공포 심리 (패닉셀 역이용)

  ELSE IF  현재가 > Band_High  AND  Sentiment Score > +70
  THEN → 매도(SELL)  ← 고평가 구간 + 환희 심리 (과열 탈출)

  ELSE
       → 관망(HOLD)

출력:
  - 매매 마커 데이터 { BUY | SELL | HOLD } + 발생 타임스탬프
```

### 4.3 UI/UX 핵심 요구사항

- **직관성 최우선**: 초보 투자자도 별도 학습 없이 매매 타점을 파악할 수 있도록 복잡한 보조지표를 배제하고, BUY/SELL 마커와 감성 게이지 중심으로 UI를 단순화한다.
- **1화면 집중 원칙 (Viewport Lock)**: 종목 선택 후 캔들 차트, 감성 게이지, 재무 요약이 **스크롤 없이** 한 뷰포트(`100vh`) 안에 표시되어야 한다. Phase 5(Next.js)에서는 `viewport-lock` 유틸과 `grid-cols-12` 레이아웃으로 강제한다.
- **Claude 스타일 다크 테마 (Phase 5)**: Zinc-950 베이스의 깊은 다크 톤 + Glassmorphism 카드(`backdrop-blur`)로 정돈된 정보 위계를 만든다.
- **색상 코딩 (시맨틱 토큰)**: 모든 UI 요소는 아래 토큰만 사용한다 (`web/tailwind.config.ts` 정의):
  - BUY / Greed → emerald (`--signal-buy`)
  - SELL / Fear → rose (`--signal-sell`)
  - HOLD / Neutral → zinc (`--signal-hold`)
  - 다이버전스 강세 → amber (`--divergence-bullish`)
  - 다이버전스 약세 → violet (`--divergence-bearish`)
- **차트 마커 근거 노출**: BUY(▲), SELL(▼), Divergence(◆) 마커 호버/클릭 시 Shadcn `Popover`/`Tooltip`으로 산출 근거(현재가·밴드·감성 점수·기여 키워드)를 즉시 표시한다.
- **게이지 직관화**: 감성 게이지는 공포/중립/탐욕 텍스트 레이블과 색상(rose/zinc/emerald)을 함께 표시하여 수치 미숙지 사용자도 즉시 상황 파악이 가능하도록 한다.
- **로딩 상태 표시**: 데이터 수집 및 분석 중에는 Shadcn `Skeleton` 또는 진행 상태 메시지를 표시하여 사용자 이탈을 방지한다.
- **국제화(i18n) 대비**: 모든 노출 텍스트는 향후 i18n 처리 가능한 형태로 작성한다 (하드코딩 한글 문자열은 텍스트 상수 또는 메시지 키로 추출).

     F-1.2           F-2.2              F-3.2              F-4.2

## 6. 용어 정의

| 용어 | 정의 |
|------|------|
| **Sentiment Score** | 뉴스 분석으로 도출한 투자 심리 수치. 범위: -100(극단적 공포) ~ +100(극단적 환희) |
| **Fundamental Band** | 기업 재무 지표 기반 적정 주가 범위(상단·하단). 과매수·과매도 판단 기준 |
| **Divergence** | 주가 추세와 감성 지수 추세가 반대로 움직이는 괴리 현상 |
| **HBM** | High Bandwidth Memory. 고대역폭 메모리. 분석 시 높은 양수 가중치를 부여하는 긍정 키워드 |
| **감산** | 반도체 생산량 감소 조치. 업계 문맥상 공급 과잉 해소 → 호재로 해석 |
| **KoNLPy** | 한국어 형태소 분석 파이썬 라이브러리 (JDK 1.8+ 필요) |
| **DART** | 금융감독원 전자공시시스템 (Data Analysis, Retrieval and Transfer System) |
