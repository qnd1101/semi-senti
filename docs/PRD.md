### 1.2 배경 및 문제 정의

기존 뉴스 감성 분석 도구와 단순 시그널 서비스는 다음 한계를 가진다.

1. **시계열 미분화**: 단일 BUY/SELL/HOLD 출력은 투자 기간(단타/스윙/중장기)에 따른 의사결정 차이를 반영하지 못한다.
2. **도메인 무지**: 반도체 업계 용어(감산, HBM, 수율, 재고 소진)를 범용 사전으로 처리해 신호 신뢰도가 낮다.
3. **설명 가능성 부족**: "왜 시그널이 발생했는지"에 대한 근거 문장이 없어 초보 투자자의 신뢰 형성이 어렵다.
4. **운영 확장성 제한**: SQLite 중심 구조는 동시성·인덱싱·관측성 측면에서 운영형 로컬 분석 서비스로 확장에 제약이 있다.

`Semi Senti`는 반도체 도메인 특화 NLP + 펀더멘털 밴드 + 가격 추세를 결합한 다층 분석으로 위 한계를 해결하고, Gemini 기반 Reasoning으로 시그널의 해석 가능성을 강화한다.

### 2.2 핵심 사용자 시나리오

#### 시나리오 1 — 관점별 신호 확인 (핵심 플로우)

```
사용자 → 대시보드 접속 (run_windows.bat / run_linux.sh → Next.js + Python API)
       → 반도체 종목 선택 (기본: 삼성전자·SK하이닉스)
       → 백그라운드: LiveDataPipeline이 pykrx 일봉 폴링 + DART 재무 갱신 → PostgreSQL UPSERT
       → 분석 엔진: 단기/중기/장기 시그널을 독립 산출
       → 화면: 3개 관점 카드(Short/Mid/Long) + 차트 마커 + 감성 게이지 + 재무 요약 표시
       → 사용자: 관점별 액션(BUY/SELL/HOLD) 차이를 비교해 의사결정
```

#### 시나리오 2 — Gemini 근거 브리핑 확인

```
사용자: 단기 BUY, 중기 HOLD, 장기 BUY가 동시에 표시된 상황 확인
      → 카드의 "근거 보기" 클릭 (Popover/상세 카드)
      → 시스템: 관점별 시그널 + 현재가 + 밴드 위치 + 감성 점수 + 핵심 키워드를 Gemini API에 전달
      → Gemini: "왜 해당 관점에서 해당 시그널이 나왔는지" 요약 근거 생성
      → 사용자: 수치+설명 기반으로 포지션 진입/유지/축소 판단
```

#### 시나리오 3 — 괴리 경보 기반 재평가

```
시스템: 주가 하락 중이나 감성 회복 + 펀더멘털 저평가 구간 감지
      → 다이버전스 마커와 관점별 상충 신호(단기 HOLD, 장기 BUY) 표출
      → 사용자: 시간축에 맞춘 분할 매수 전략 재설정
```

#### F-2.1 / F-2.2 반도체 특화 NLP 감성 스코어링

**처리 파이프라인:**

1. **형태소 분석 (Tokenize)**  
   KoNLPy로 뉴스 본문·헤드라인에서 명사·형용사 추출
2. **키워드 매칭 (Matching)**  
   반도체 특화 사전과 대조해 가중치 부여 (`감산`, `수율 개선`, `HBM`, `재고 부담` 등)
3. **점수 산출 (Calculation)**  
   \[
   RawScore = \sum_{i=1}^{n} w_i
   \]
4. **정규화 (Normalization)**  
   `-100 ~ +100`으로 정규화 (`-100` 극단 공포, `0` 중립, `+100` 극단 탐욕)
5. **저장 (Persist)**  
   종목·시각 단위 감성 지수와 상위 키워드를 PostgreSQL에 기록

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-2.1.1 | KoNLPy로 뉴스 텍스트에서 핵심 명사·형용사를 추출한다 | P1 |
| F-2.2.1 | 반도체 특화 사전 기반 가중치 점수를 산출한다 | P1 |
| F-2.2.2 | 최종 감성 점수(-100~+100)와 핵심 키워드를 DB에 저장한다 | P1 |

#### F-3.1 / F-3.2 / F-3.3 다중 관점 시그널 및 근거 생성

**F-3 분석 아키텍처 흐름 (텍스트 시나리오):**

```
[수집 계층]
  pykrx 일봉 + DART 재무 + 네이버 뉴스
      ↓
[정제/저장 계층]
  PostgreSQL UPSERT (financials, news, sentiment_scores, fundamentals)
      ↓
[분석 계층]
  (1) 펀더멘털 밴드 산출 (F-3.1)
  (2) 단기/중기/장기 시그널 산출 (F-3.2)
  (3) Gemini Reasoning 생성 (F-3.3)
      ↓
[제공 계층]
  FastAPI Snapshot API → Next.js 대시보드 (카드/마커/Popover)
```

**매매 시그널 산출 로직 (관점별 의사코드):**

```text
입력 공통:
  P_now            = 현재가
  Band_low, Band_high = 펀더멘털 밴드 하단/상단
  S_news           = 실시간 뉴스 감성 점수 (-100 ~ +100)
  K_trend          = 핵심 키워드 모멘텀 점수 (-100 ~ +100)
  Price_D/W/M      = 일봉/주봉/월봉 추세 점수 (-100 ~ +100)
  Cycle_score      = 업황 사이클 점수 (-100 ~ +100)
  Inventory_turnover_z = 재고자산 회전율 표준화 점수

파생 변수:
  Valuation_pos = clamp((P_now - Band_mid) / (Band_high - Band_low), -1, 1)
  # -1에 가까울수록 저평가, +1에 가까울수록 고평가

단기 관점 (1일~2주):
  Short_score =
    0.45 * S_news +
    0.35 * Price_D +
    0.20 * (-Valuation_pos * 100)

중기 관점 (2주~3개월):
  Mid_score =
    0.25 * S_news +
    0.30 * Price_W +
    0.35 * (-Valuation_pos * 100) +
    0.10 * K_trend

장기 관점 (3개월~12개월+):
  Long_score =
    0.10 * S_news +
    0.20 * Price_M +
    0.35 * (-Valuation_pos * 100) +
    0.20 * Cycle_score +
    0.15 * Inventory_turnover_z

판정 규칙 (관점별 독립 적용):
  if score >= +25  then BUY
  if score <= -25  then SELL
  else HOLD
```

**Gemini Reasoning 파이프라인:**

```text
입력 패킷:
  - short/mid/long 시그널 및 score
  - 현재가, 밴드 위치(저평가/중립/고평가)
  - 최신 감성 점수, 핵심 키워드 Top-N
```

```
[데이터 수집] → [NLP 감성 분석] → [관점별 시그널] → [Gemini 근거 생성] → [대시보드 시각화]
    F-1.1           F-2.1              F-3.2               F-3.3               F-4.1
    F-1.2           F-2.2              F-3.1                                   F-4.2
    F-1.3                                                                        F-4.3
```

**MVP 포함 기능 체크리스트 (P1):**

- [x] DART 재무 + pykrx 주가 수집 및 PostgreSQL 캐싱/UPSERT
- [x] 네이버 뉴스 수집·전처리 및 감성 점수(-100~+100) 산출
- [x] 단기/중기/장기 3관점 시그널 도출(관점별 가중치 모델)
- [x] Gemini API 기반 투자 판단 근거(Reasoning) 생성 및 UI 노출
- [x] 캔들 차트(일·주·월·년) + 관점별 마커 + 감성 게이지 + 재무 요약 1화면 구성

**MVP 제외 기능 (Post-MVP):**

- [ ] 텔레그램 알림(F-5.1)
- [ ] 다이버전스 고도화 탐지(F-2.3)
- [ ] 업황 사이클 심화 분석(F-2.4)

## 6. 용어 정의

| 용어 | 정의 |
|------|------|
| **Sentiment Score** | 뉴스 분석으로 도출한 투자 심리 수치. 범위: -100(극단 공포) ~ +100(극단 환희) |

| **Short/Mid/Long Signal** | 투자 시간축(단기/중기/장기)별로 독립 산출되는 BUY/SELL/HOLD 시그널 |

| **Gemini API** | 구조화 프롬프트를 입력받아 관점별 투자 판단 근거 문장을 생성하는 생성형 AI API |
| **Reasoning** | 시그널 도출 근거를 사람이 이해 가능한 문장으로 요약한 설명 텍스트 |

| **DART** | 금융감독원 전자공시시스템 재무 API |
| **pykrx** | KRX·네이버 기반 일봉 OHLCV 수집 Python 라이브러리 |

| **LiveDataPipeline** | FastAPI 기동 시 초기 동기화와 주기 폴링을 수행하는 백그라운드 수집 파이프라인 |
