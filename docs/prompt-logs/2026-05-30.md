> ### 🚨 핵심 변경 요구사항
> 
> 1. 매매 시그널 로직 고도화 (단기/중기/장기 관점 세분화)

>    - 투자자의 시계열에 맞춰 [단기 관점], [중기 관점], [장기 관점]별로 매수/매도/관망 여부를 각각 독립적으로 산출하고 대시보드에 표시해야 합니다.
>    - 각 관점별로 어떤 데이터(예: 단기는 실시간 뉴스 감성 + 일봉 주가, 중기는 주봉 + 펀더멘털 밴드, 장기는 월봉 + 재고자산 회전율 및 업황 사이클 등)를 가중치로 둘 것인지 로직의 기준을 기능 명세에 명확히 정의해주세요.
> 
> 2. UI/UX 컨셉 명확화 (Claude 스타일 다크 모드 고도화)
>    - UI 핵심 요구사항에서 'Claude 스타일 화이트 테마'를 더욱 구체화해주세요.
>    - Zinc-950 기반의 깊은 배경색, Glassmorphism(`backdrop-blur`), 세련되고 절제된 타이포그래피, Shadcn UI와 Lucide React를 적극 활용한 미니멀하고 고급스러운 대시보드 레이아웃(Viewport Lock)을 묘사해야 합니다.
> 
> 3. 데이터베이스 마이그레이션 (SQLite → 로컬 PostgreSQL)
>    - 로컬 환경의 개발 및 운영 편의성을 위해 가볍게 쓰던 SQLite를 제거하고, '로컬 PostgreSQL'로 전환합니다.
>    - 기술 스택, 비기능 요구사항, 데이터 적재(UPSERT) 관련 명세에서 SQLite 언급을 모두 PostgreSQL로 수정하고, 로컬 PostgreSQL 환경 구성에 필요한 제약사항이나 인덱스 설계 방향이 있다면 녹여내 주세요.
> 
> 4. Gemini API 기반의 매매 근거(Reasoning) 자동 생성 기능 추가
>    - 단순 규칙(Rule-based) 기반 시그널 출력을 넘어, Gemini API를 연동합니다.
>    - 산출된 단/중/장기 시그널, 현재 주가 데이터, 펀더멘털 밴드 위치, 그리고 KoNLPy로 분석된 실시간 뉴스 감성 점수와 핵심 키워드를 프롬프트로 구성하여 Gemini API에 전달합니다.
>    - Gemini는 이 데이터들을 종합 분석하여 "왜 지금 이 시그널이 도출되었는지"에 대한 객관적이고 직관적인 '투자 판단 근거(Reasoning)' 텍스트를 생성하며, UI(Popover 또는 상세 카드)에 노출합니다. 이 파이프라인과 API 키(`.env` 의 `GEMINI_API_KEY`) 의존성을 명세에 추가하세요.
