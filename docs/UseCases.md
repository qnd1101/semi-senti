### UC-01: 종목 분석 조회

| 항목 | 내용 |
|------|------|
| **UC ID** | UC-01 |
| **유즈케이스명** | 종목 분석 조회 |
| **액터** | 일반 투자자 (A-01) |
| **목표** | 원하는 반도체 종목을 선택하여 재무 요약 정보와 분석 결과를 한 화면에서 확인한다 |
| **사전 조건** | - 시스템이 정상 구동 중이다<br>- SQLite DB에 해당 종목의 캐시 데이터가 존재하거나, 외부 API 연결이 가능하다 |
| **사후 조건** | 선택한 종목의 캔들 차트, 감성 게이지, 재무 요약 지표가 대시보드에 렌더링된다 |

**기본 흐름 (Main Flow):**

```
1. 투자자가 Streamlit 대시보드에 접속한다.
2. 종목 선택 UI(드롭다운 또는 검색창)에서 분석할 종목을 선택한다.
   예: '삼성전자', 'SK하이닉스'
3. 시스템은 SQLite 캐시에 해당 종목의 유효한 데이터가 있는지 확인한다.
4. 캐시 히트 시: DB에서 주가·재무·감성 데이터를 즉시 로드한다.
5. 분석 엔진이 최신 감성 점수와 펀더멘털 밴드를 산출한다.
6. TradingView API로 캔들 차트를 렌더링하고,
   BUY/SELL/HOLD 시그널 마커를 해당 타임스탬프에 표시한다.
7. 감성 게이지 컴포넌트가 현재 감성 점수를 시각화한다.
8. 재무 요약(매출액·영업이익·PER·PBR·EPS)을 대시보드 패널에 출력한다.
9. 투자자는 한 화면에서 모든 정보를 확인한다.
```

**대안 흐름 (Alternative Flow):**

```
4a. 캐시 미스 (데이터 없음 또는 TTL 만료):
    4a-1. 시스템이 외부 API(DART, 네이버, yfinance)에 실시간 요청을 보낸다.
    4a-2. 수집된 데이터를 전처리(HTML 제거, 단위 통일)하여 SQLite에 캐싱한다.
    4a-3. 기본 흐름 5번으로 복귀한다.

2a. 투자자가 목록에 없는 종목명을 직접 입력:
    2a-1. 시스템이 "분석 대상 종목이 아닙니다" 메시지를 표시한다.
    2a-2. 관리자에게 종목 추가를 요청하도록 안내한다.
```

**예외 흐름 (Exception Flow):**

```
E1. 외부 API 타임아웃 발생:
    E1-1. 시스템이 캐시된 이전 데이터를 폴백으로 로드한다.
    E1-2. 대시보드 상단에 "데이터가 [최종 갱신 시각] 기준입니다" 경고를 표시한다.
    E1-3. 기본 흐름 5번으로 복귀한다.

E2. SQLite DB 접근 오류:
    E2-1. "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요" 메시지를 표시한다.
    E2-2. 유즈케이스 종료.
```

### UC-02: 매매 시그널 확인

| 항목 | 내용 |
|------|------|
| **UC ID** | UC-02 |
| **유즈케이스명** | 매매 시그널 확인 |
| **액터** | 일반 투자자 (A-01) |
| **목표** | 선택한 종목의 차트에서 BUY/SELL/HOLD 시그널 마커를 확인하여 매매 판단을 보조한다 |
| **사전 조건** | - UC-01이 완료되어 대시보드가 렌더링된 상태이다<br>- 분석 엔진이 시그널을 산출한 기록이 DB에 존재한다 |
| **사후 조건** | 투자자가 차트 위에서 각 시그널의 발생 위치와 근거를 확인한다 |

**기본 흐름 (Main Flow):**

```
1. 대시보드에서 TradingView 캔들 차트가 렌더링된다.
2. SignalLogic이 DB에서 해당 종목의 시그널 이력을 조회한다.
3. 시그널 마커가 차트의 해당 날짜/시간 위치에 표시된다:
   - BUY  마커: ▲ 녹색 화살표
   - SELL 마커: ▼ 적색 화살표
   - HOLD: 마커 미표시
4. 투자자가 마커를 클릭(또는 호버)하면 시그널 발생 근거 팝업이 표시된다:
   예: "감성 점수: -82 / 현재가: 밴드 하단 대비 -12%"
5. 투자자는 현재 시점의 최신 시그널을 확인하여 매매를 판단한다.
```

**시그널 산출 로직 참조:**

```
조건 1 (BUY):
  현재가 < 펀더멘털 밴드 하단  AND  감성 점수 < -70
  → 저평가 구간 + 공포 심리 (패닉셀 역이용 전략)

조건 2 (SELL):
  현재가 > 펀더멘털 밴드 상단  AND  감성 점수 > +70
  → 고평가 구간 + 환희 심리 (과열 탈출 전략)

기본값 (HOLD):
  위 두 조건 모두 미충족
```

**대안 흐름:**

```
3a. 조회 기간 내 발생한 시그널이 없는 경우:
    3a-1. "선택 기간에 매매 시그널이 없습니다. 현재 구간: HOLD" 메시지를 표시한다.
```

**예외 흐름:**

```
E1. TradingView API 렌더링 실패:
    E1-1. "차트를 불러올 수 없습니다" 메시지를 표시하고,
          텍스트 기반 시그널 요약 테이블로 대체 표시한다.
```

### UC-03: 감성 게이지 조회

| 항목 | 내용 |
|------|------|
| **UC ID** | UC-03 |
| **유즈케이스명** | 감성 게이지 조회 |
| **액터** | 일반 투자자 (A-01) |
| **목표** | 현재 시장 심리(공포/중립/탐욕)를 직관적인 게이지로 확인하고, 주요 반도체 키워드 트렌드를 파악한다 |
| **사전 조건** | - NLP 엔진이 최근 뉴스 기사에 대해 감성 점수를 산출한 상태이다 |
| **사후 조건** | 게이지에 현재 감성 점수와 상태 레이블, 핵심 키워드가 표시된다 |

**기본 흐름 (Main Flow):**

```
1. 대시보드 감성 게이지 패널이 DB에서 최신 감성 점수를 로드한다.
2. SentimentGauge 컴포넌트가 점수에 따라 게이지를 렌더링한다:

   점수 범위         | 상태 레이블  | 색상
   -100 ~ -34       | 공포(Fear)   | 파랑(Blue)
   -33  ~ +33       | 중립(Neutral)| 회색(Gray)
   +34  ~ +100      | 탐욕(Greed)  | 빨강(Red)

3. 감성 점수 산출에 기여한 상위 키워드 목록이 게이지 하단에 표시된다.
   예: HBM(+23), 감산(+15), 재고(-18), 수율(+9) ...
4. 투자자는 현재 시장 심리를 수치와 레이블로 동시에 파악한다.
5. 게이지는 설정된 주기(5분)마다 자동으로 갱신된다.
```

**NLP 감성 점수 산출 흐름 (시스템 내부):**

```
Step 1 [Tokenize]   KoNLPy로 뉴스 본문에서 명사·형용사 추출
Step 2 [Matching]   추출된 단어 ↔ 반도체 특화 사전 비교
                    (감산·HBM·수율·재고·수요·공급 등)
Step 3 [Score]      Raw Score = Σ(매칭 키워드 가중치)
Step 4 [Normalize]  정규화 → -100 ~ +100 범위 변환
Step 5 [Store]      일자별·종목별 감성 지수를 SQLite에 저장
```

**대안 흐름:**

```
1a. 최근 24시간 내 수집된 뉴스가 없는 경우:
    1a-1. "최신 뉴스 데이터가 없습니다. [마지막 갱신: N시간 전]" 안내를 표시한다.
    1a-2. 캐시된 이전 감성 점수를 회색 처리하여 표시한다.
```

### UC-04: 다이버전스 시그널 확인

| 항목 | 내용 |
|------|------|
| **UC ID** | UC-04 |
| **유즈케이스명** | 다이버전스 시그널 확인 |
| **액터** | 일반 투자자 (A-01) |
| **목표** | 주가 추세와 감성 지수 추세의 괴리를 차트에서 확인하여 추세 반전 가능성을 선제적으로 파악한다 |
| **사전 조건** | - DivergenceDetector가 괴리 현상을 탐지한 기록이 DB에 존재한다 |
| **사후 조건** | 차트에 다이버전스 마커가 표시되고, 투자자는 반전 가능성을 인지한다 |

**기본 흐름 (Main Flow):**

```
1. DivergenceDetector가 주가 N일 추세와 감성 지수 N일 추세를 비교한다.
2. 두 추세가 반대 방향으로 움직이는 구간을 탐지한다:

   유형 A (강세 다이버전스):
     주가 하락 추세  +  감성 지수 상승 추세
     → '기회(Opportunity)' 마커 (황색 ◆) 표시

   유형 B (약세 다이버전스):
     주가 상승 추세  +  감성 지수 하락 추세
     → '주의(Caution)' 마커 (보라색 ◆) 표시

3. 마커를 차트 해당 구간에 표시한다.
4. 투자자가 마커 호버 시 근거 팝업이 표시된다:
   예: "주가 -5.2% vs 감성 지수 +18pt (3일 기준)"
5. 투자자는 포지션 재검토 여부를 판단한다.
```

**대안 흐름:**

```
1a. 탐지 기간 내 다이버전스가 없는 경우:
    1a-1. 다이버전스 마커 없이 기본 차트만 표시된다.
```

### UC-05: 텔레그램 알림 수신

| 항목 | 내용 |
|------|------|
| **UC ID** | UC-05 |
| **유즈케이스명** | 텔레그램 알림 수신 |
| **액터** | 일반 투자자 (A-01), 텔레그램 API (A-06) |
| **목표** | 대시보드를 열지 않은 상태에서도 중요 매매 시그널 발생 시 즉각 알림을 수신한다 |
| **사전 조건** | - 투자자가 텔레그램 봇과 연동되어 있다<br>- NotificationManager가 활성화된 상태이다 |
| **사후 조건** | 투자자의 텔레그램에 시그널 알림 메시지가 전송된다 |

**기본 흐름 (Main Flow):**

```
1. SignalLogic이 BUY 또는 SELL 시그널을 새로 산출한다.
2. NotificationManager가 시그널 발생 이벤트를 수신한다.
3. 텔레그램 API를 통해 아래 형식의 메시지를 발송한다:

   ─────────────────────────────────
   🔔 [Semi Senti] 매매 시그널 발생
   ─────────────────────────────────
   종목   : SK하이닉스 (000660)
   시그널 : 🟢 BUY
   현재가 : 128,000원
   밴드   : 하단 131,500원 대비 -2.7%
   감성   : -82 (공포 구간)
   시각   : 2026-05-02 14:32
   ─────────────────────────────────

4. 투자자가 알림을 수신하고 대시보드 접속 여부를 결정한다.
```

**대안 흐름 A — 감성 급변 알림:**

```
1a. 감성 점수가 설정된 임계값을 급격히 초과(예: ±30pt 이상 급변)한 경우:
    1a-1. NotificationManager가 경고 알림을 발송한다:
          예: "⚠️ SK하이닉스 감성 지수 급락: +12 → -45 (1시간 내)"
```

**예외 흐름:**

```
E1. 텔레그램 API 전송 실패:
    E1-1. 최대 3회 재시도 후 실패 시 시스템 로그에 오류를 기록한다.
    E1-2. 투자자가 다음 대시보드 접속 시 "미발송 알림 N건" 배지를 표시한다.
```

| 항목 | 내용 |

| **유즈케이스명** | 분석 종목 관리 |
| **액터** | 관리자 (A-02) |
| **목표** | 시스템이 분석할 반도체 종목 목록을 추가·수정·삭제하여 서비스 커버리지를 관리한다 |
| **사전 조건** | - 관리자가 Admin System에 로그인한 상태이다 |
| **사후 조건** | 변경된 종목 목록이 즉시 대시보드 종목 선택 UI에 반영된다 |

**기본 흐름 (Main Flow):**

```
1. 관리자가 Admin System > 종목 관리 메뉴에 접속한다.
2. 현재 등록된 종목 목록이 테이블로 표시된다.
3. 관리자가 [종목 추가] 버튼을 클릭하고 종목 코드를 입력한다.
   예: 종목명 "DB하이텍", 종목 코드 "000990"
4. 시스템이 yfinance API로 해당 종목 코드의 유효성을 검증한다.
5. 유효성 통과 시 DB Stocks 테이블에 신규 종목을 등록한다.
6. DataCollector가 신규 종목의 초기 데이터 수집을 스케줄링한다.
7. 대시보드 종목 목록에 즉시 반영된다.
```

**대안 흐름 A — 종목 삭제:**

```
3a. 관리자가 기존 종목 행의 [삭제] 버튼을 클릭한다.
3a-1. "해당 종목의 분석 데이터도 함께 삭제됩니다. 계속하시겠습니까?" 확인 팝업 표시.
3a-2. 확인 클릭 시 DB에서 종목 및 관련 데이터를 삭제한다.
```

**예외 흐름:**

```
E1. 입력한 종목 코드가 yfinance에서 조회되지 않는 경우:
    E1-1. "유효하지 않은 종목 코드입니다. 다시 확인해 주세요" 오류 메시지를 표시한다.
```

### UC-07: 데이터 수동 갱신 (관리자)

| 항목 | 내용 |
|------|------|
| **UC ID** | UC-07 |
| **유즈케이스명** | 데이터 수동 갱신 |
| **액터** | 관리자 (A-02) |
| **목표** | API 스케줄 외에도 특정 종목의 데이터를 즉시 갱신하여 최신 분석 결과를 확보한다 |
| **사전 조건** | - API 잔여 쿼터가 충분하다<br>- 관리자가 Admin System에 접속한 상태이다 |
| **사후 조건** | 선택한 종목의 최신 데이터가 DB에 업데이트되고, 감성 점수 및 시그널이 재산출된다 |

**기본 흐름 (Main Flow):**

```
1. 관리자가 Admin System > 모니터링 화면에서 특정 종목을 선택한다.
2. [즉시 갱신] 버튼을 클릭한다.
3. 시스템이 현재 API 잔여 쿼터를 확인한다.
4. DataCollector가 해당 종목의 데이터를 외부 API에서 실시간 수집한다:
   - DART: 최신 재무 데이터
   - 네이버 뉴스 API: 최근 기사
   - yfinance: 최신 주가
5. 수집된 데이터를 전처리하여 SQLite 캐시를 업데이트한다.
6. SentimentEngine이 감성 점수를 재산출한다.
7. SignalLogic이 매매 시그널을 재도출한다.
8. "갱신 완료: [종목명] 데이터 업데이트됨 (HH:MM)" 완료 메시지를 표시한다.
```

**예외 흐름:**

```
E1. API 일일 쿼터 초과:
    E1-1. "오늘의 API 호출 한도에 도달했습니다. 내일 자동으로 갱신됩니다" 안내를 표시한다.
    E1-2. 캐시된 데이터를 유지하고 강제 갱신을 차단한다.
```
